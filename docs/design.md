## 0. 디자인 공통 원칙

- **정보 노출 최소화**: 목록 화면에서는 이름/썸네일 등 최소 정보만 노출, 상세 정보는 상세 페이지에서만 확인.
- **일관된 헤더/네비게이션**: 모든 페이지 상단에 공통 GNB(Global Navigation Bar) 배치 — 로고, 검색, 상품등록 버튼, 알림, 마이페이지 진입.
- **신고/차단 상태의 명시적 표시**: 차단된 상품, 휴면 계정은 회색조 처리 + 배지(badge)로 구분하여 사용자가 오인하지 않도록 함.
- **모바일 우선 반응형**: 와이어프레임은 모바일 기준(360~420px)으로 작성, 데스크톱은 좌우 여백 확장 + 채팅 패널이 사이드로 이동하는 구조.

### 공통 컴포넌트 목록

| 컴포넌트 | 설명 |
|---|---|
| `GNB` | 상단 고정 네비게이션 바 (로고, 검색창, 등록 버튼, 알림 아이콘, 프로필 아이콘) |
| `BottomTabBar` (모바일) | 홈 / 채팅 / 등록 / 마이페이지 4탭 |
| `Avatar` | 유저 프로필 이미지, 휴면 계정 시 회색 처리 + 아이콘 오버레이 |
| `Badge` | 상태 표시 배지 (예: `판매중`, `예약중`, `거래완료`, `신고됨`, `휴면계정`) |
| `Toast` | 액션 결과 알림 (저장 완료, 신고 접수 등) |
| `ConfirmModal` | 삭제/신고/로그아웃 등 파괴적 액션 확인 모달 |
| `EmptyState` | 목록 없음, 검색결과 없음 등 빈 상태 안내 |

## 1. 회원가입 페이지

### (1) 주요 컴포넌트 구성

- `AuthLayout` (중앙 정렬 카드형 레이아웃)
- `TextInput` × N: 이메일, 비밀번호, 비밀번호 확인, 닉네임
- `TextArea`: 한줄 소개 (선택, 마이페이지에서도 수정 가능)
- `PasswordStrengthHint`: 비밀번호 규칙 안내 텍스트
- `SubmitButton`: "가입하기" (유효성 통과 전 비활성화)
- `InlineError`: 필드별 실시간 유효성 에러 메시지
- `LinkToLogin`: "이미 계정이 있으신가요? 로그인"

### (2) 레이아웃 (ASCII 와이어프레임)

```
┌──────────────────────────────────┐
│           [로고/서비스명]          │
│                                    │
│   회원가입                         │
│  ┌──────────────────────────────┐ │
│  │ 이메일                        │ │
│  └──────────────────────────────┘ │
│  ┌──────────────────────────────┐ │
│  │ 비밀번호                      │ │
│  └──────────────────────────────┘ │
│  ┌──────────────────────────────┐ │
│  │ 비밀번호 확인                  │ │
│  └──────────────────────────────┘ │
│  ┌──────────────────────────────┐ │
│  │ 닉네임                        │ │
│  └──────────────────────────────┘ │
│  ┌──────────────────────────────┐ │
│  │ 한줄 소개 (선택)                │ │
│  └──────────────────────────────┘ │
│                                    │
│  ┌──────────────────────────────┐ │
│  │        가입하기 (버튼)          │ │
│  └──────────────────────────────┘ │
│                                    │
│      이미 계정이 있으신가요? 로그인   │
└──────────────────────────────────┘
```

### (3) 사용자 인터랙션 흐름

1. 사용자가 각 입력 필드에 값을 입력 → 필드 blur 시 실시간 유효성 검사 (이메일 형식, 비밀번호 규칙, 비밀번호 일치 여부, 닉네임 중복 등) → 실패 시 `InlineError` 표시.
2. 모든 필드가 유효하면 "가입하기" 버튼 활성화.
3. 버튼 클릭 → 로딩 스피너 표시 → 성공 시 `Toast("가입이 완료되었습니다")` 후 로그인 페이지 또는 메인 페이지로 자동 이동(자동 로그인 정책에 따라 분기).
4. 실패(이메일 중복 등) 시 폼 상단에 에러 배너 표시, 입력값은 유지.
5. "이미 계정이 있으신가요? 로그인" 클릭 → 로그인 페이지로 이동.

## 2. 로그인 페이지

### (1) 주요 컴포넌트 구성

- `AuthLayout` (회원가입과 동일한 카드형 레이아웃 재사용)
- `TextInput` × 2: 이메일, 비밀번호
- `SubmitButton`: "로그인"
- `InlineError` / `FormErrorBanner`: 로그인 실패 사유 표시 (계정 없음, 비밀번호 불일치, **휴면 계정 안내** 등)
- `LinkToSignup`: "계정이 없으신가요? 회원가입"

### (2) 레이아웃 (ASCII 와이어프레임)

```
┌──────────────────────────────────┐
│           [로고/서비스명]          │
│                                    │
│   로그인                          │
│  ┌──────────────────────────────┐ │
│  │ 이메일                        │ │
│  └──────────────────────────────┘ │
│  ┌──────────────────────────────┐ │
│  │ 비밀번호                      │ │
│  └──────────────────────────────┘ │
│                                    │
│  ┌──────────────────────────────┐ │
│  │          로그인 (버튼)          │ │
│  └──────────────────────────────┘ │
│                                    │
│      계정이 없으신가요? 회원가입     │
└──────────────────────────────────┘
```

### (3) 사용자 인터랙션 흐름

1. 이메일/비밀번호 입력 후 "로그인" 클릭.
2. 성공 시 메인 페이지로 이동.
3. 실패 시 사유별 분기 메시지:
   - 계정 없음/비밀번호 불일치 → "이메일 또는 비밀번호가 올바르지 않습니다."
   - **휴면 계정** → 별도 안내 배너: "장기간 미접속으로 휴면 처리된 계정입니다. 본인 인증 후 계정을 활성화해주세요." + `계정 활성화` 버튼 노출 (활성화 플로우로 분기).
   - **신고 누적으로 정지된 계정** → "신고 누적으로 이용이 제한된 계정입니다. 문의: ..." 안내.
4. "계정이 없으신가요? 회원가입" 클릭 → 회원가입 페이지로 이동.

## 3. 메인 페이지 (전체 상품 조회 + 실시간 전체채팅)

가장 복잡한 화면으로, 상품 목록과 실시간 전체채팅이 하나의 화면에 공존합니다. 모바일에서는 탭 전환, 데스크톱에서는 좌/우 분할 레이아웃을 사용합니다.

### (1) 주요 컴포넌트 구성

- `GNB`
- `CategoryFilterBar`: 카테고리/정렬 필터 (선택)
- `ProductListPanel`
  - `ProductCard` × N: **상품명만 노출** (썸네일 + 이름, 가격은 옵션), 클릭 시 상세 페이지 이동
  - `ProductCard(blocked)`: 신고 누적/차단된 상품은 흐림 처리 + `Badge("신고로 노출 제한됨")`, 클릭 비활성화 또는 안내 모달만 노출
  - `InfiniteScrollLoader` / `Pagination`
  - `EmptyState`: 검색/필터 결과 없음
- `GlobalChatPanel` (실시간 전체 채팅)
  - `ChatMessageList`: 발신자 닉네임(휴면 계정은 회색 처리) + 메시지 + 시각
  - `ChatInput` + `SendButton`
  - `ChatSystemNotice`: 접속자 수, 공지 등
- `TabSwitcher` (모바일 전용: [상품목록] / [전체채팅] 탭)
- `FloatingActionButton`: "+ 상품 등록" (모바일 우하단 고정)

### (2) 레이아웃 (ASCII 와이어프레임)

**모바일 (탭 전환형)**

```
├──────────────────────────────────┤
│ 전체 | 의류 | 전자 | 가구 | ...     │  ← CategoryFilterBar
├──────────────────────────────────┤
│ ┌───────┐ ┌───────┐ ┌───────┐    │
│ │ 사진   │ │ 사진   │ │ 사진   │    │
│ │ 상품명A │ │ 상품명B │ │ 상품명C │    │

│ │ 사진   │ │▓▓▓▓▓▓▓│ │ 사진   │    │
```

### (3) 사용자 인터랙션 흐름

1. 진입 시 상품 목록(그리드)과 전체채팅이 동시 로드. 모바일은 기본 [상품목록] 탭 활성.
2. `ProductCard`는 **상품명(+ 썸네일)만 노출**. 카드 클릭 → 상품 상세 페이지로 라우팅.
3. 차단/신고 처리된 상품 카드는 흐림 처리 + `신고로 노출 제한됨` 배지, 클릭 시 상세 이동 대신 안내 모달("신고 접수로 검토 중인 상품입니다") 노출.
4. 카테고리 필터/검색 입력 시 목록 실시간 갱신, 결과 없으면 `EmptyState` 노출.
5. 무한 스크롤로 하단 도달 시 다음 페이지 로드.
6. [전체채팅] 탭(모바일) 또는 우측 패널(데스크톱): 메시지 입력 후 전송 버튼/Enter → 채팅 리스트 하단에 실시간 반영, 자동 스크롤.
7. 휴면 계정 사용자가 작성한 채팅은 닉네임을 회색 처리하고 `(휴면)` 접두 표시하여 활동 신뢰도를 구분.
8. 채팅 내 닉네임 클릭 → 유저 프로필 조회 페이지로 이동.
9. 우하단 FAB 또는 GNB "+ 상품등록" 클릭 → 새 상품 등록 페이지로 이동.

## 4. 새 상품 등록 페이지

### (1) 주요 컴포넌트 구성

- `GNB` (뒤로가기 포함)
- `ImageUploader`: 다중 이미지 업로드, 썸네일 미리보기, 대표이미지 지정, 삭제
- `TextInput`: 상품명
- `PriceInput`: 가격 (숫자, 원화 포맷)
- `CategorySelect`: 카테고리 드롭다운
- `TextArea`: 상품 설명
- `RadioGroup`: 거래 상태 (판매중 — 등록 시 기본값)
- `RadioGroup` (선택): 거래 방식 (직거래/택배 등, 확장 옵션)
- `SubmitButton`: "등록하기" (필수값 검증 후 활성화)
- `ConfirmModal`: 뒤로가기 시 "작성 중인 내용이 사라집니다" 확인

1. 이미지 업로드(최소 1장 필수) → 썸네일 미리보기, 길게 누르거나 X 버튼으로 삭제/순서 변경, 첫 번째 이미지가 대표 이미지.
2. 상품명/가격/카테고리/설명 입력. 필수 항목 미입력 시 "등록하기" 비활성화 + 미입력 필드 하이라이트.
3. 뒤로가기(←) 클릭 시 입력값 있으면 `ConfirmModal`로 이탈 확인.
4. "등록하기" 클릭 → 유효성 검증 → 로딩 → 성공 시 `Toast("상품이 등록되었습니다")` 후 해당 상품 상세 페이지로 이동.
5. 등록 직후 메인 페이지 목록 최상단에 신규 등록 상품 반영(정렬 기준: 최신순 기준).

## 8. 신고 페이지 (상품신고 / 유저신고)

상품 상세, 유저 프로필, 채팅 등 여러 진입점에서 호출되는 공용 페이지(또는 모달)이며, 신고 대상/유형에 따라 폼이 사전 채워집니다.

### (1) 주요 컴포넌트 구성

```
┌──────────────────────────────────┐
│                                    │
│              ✔                    │
│         신고가 접수되었습니다        │
│    검토 후 처리 결과를 안내드립니다.  │
│                                    │
│        [ 확인 ]                    │
└──────────────────────────────────┘
```

### (3) 사용자 인터랙션 흐름

1. 상품 상세의 "상품 신고하기", 유저 프로필의 "이 사용자 신고하기" 등에서 진입 → 대상 정보가 `ReportTargetSummary`에 자동 채워짐.
2. 사유 카테고리(라디오) 선택 + 상세 사유 텍스트 입력(필수, 최소 글자수 미달 시 `InlineError`와 함께 제출 버튼 비활성 유지).
3. 증빙 자료(선택) 첨부 가능.
4. "신고 접수" 클릭 → `ConfirmModal`로 최종 확인(허위 신고 제재 경고 문구 포함) → 확인 시 제출.
5. 제출 성공 → `SuccessNotice` 전체 화면 안내 → 확인 클릭 시 신고 이전 화면(상품 상세/프로필)으로 복귀.
6. 동일 대상에 대해 이미 신고 이력이 있는 경우 "이미 신고 접수된 대상입니다" 안내와 함께 중복 제출 방지(선택적 정책, 기획 확정 필요 항목으로 별도 표기).

## 9. 공통 상태 처리 (신고/차단/휴면)

여러 화면에 걸쳐 일관되게 적용되어야 하는 상태 표시 규칙을 정리합니다.

| 상태 | 목록/카드 노출 | 상세 진입 | 배지 텍스트 | 비고 |
|---|---|---|---|---|
| 정상 상품 | 정상 노출 | 가능 | `판매중` / `예약중` / `거래완료` | - |
| 신고 누적/차단 상품 | 흐림 처리, 클릭 비활성(안내 모달만) | 등록자 본인만 가능(배너 노출) | `신고로 노출 제한됨` | 관리자 검토 전까지 타인 노출 차단 |
| 정상 유저 | 정상 노출 | 가능 | - | - |
| 휴면 계정 유저 | 아바타/닉네임 회색 처리, `(휴면)` 표기 | 프로필 열람은 가능, 신규 채팅 시작 불가 | `휴면계정` | 로그인 시도 시 별도 활성화 안내(로그인 페이지 참고) |
| 정지/제재 유저(신고 누적) | 목록/채팅에서 닉네임 비식별 처리 검토 | 프로필 진입 시 제한 안내 | `이용제한` | 정책 상세는 별도 확정 필요 |

**적용 화면**: 메인 페이지(상품 카드, 채팅 닉네임), 상품 상세(판매자 정보, 차단 배너), 마이페이지(내 상품 상태 배지), 유저 프로필(휴면 배너), 로그인(휴면/제재 안내).

## 남은 확인 필요 사항 (기획/백엔드 협의 필요, 디자인 범위 외)

- 신고 누적 임계치(몇 건부터 자동 차단되는지) 및 차단 해제 프로세스
- 휴면 계정 판정 기준(미접속 기간)과 활성화 인증 방식
- 채팅 메시지 모더레이션(욕설 필터 등) 적용 여부
- 중복 신고 방지 정책 확정
